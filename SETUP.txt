Connect this repo to Lovable
============================

1. Create a Supabase project at https://supabase.com
2. Run migrations in order from supabase/migrations/ in the SQL Editor
3. Create an admin user in Auth, then run supabase/scripts/bootstrap-super-admin.sql
4. Deploy Edge Functions:
   supabase functions deploy start-claim
   supabase functions deploy start-registration
   supabase functions deploy import-preview
   supabase functions deploy import-commit
   supabase functions deploy notify-registrant
5. Copy .env.example to .env and set VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY
6. npm install && npm run dev
7. In Lovable: connect your Supabase project and import/paste this codebase
   - Prompt Lovable to use existing schema exactly (do not invent new tables)
   - Start with auth shell only, then iterate slices per plan

Lovable prompt starter
----------------------
Use the existing Supabase schema: profiles, alumni_members, approval_requests,
import_batches, imported_records. Roll number is immutable for alumni. No directory
access until approved. Email-only OTP/magic link. Premium fields are metadata only.
Alumni can hide from directory via is_directory_visible.

Performance testing
-------------------
npm run seed:synthetic   # requires SUPABASE_URL + SUPABASE_SERVICE_ROLE_KEY
npm run audit:security
